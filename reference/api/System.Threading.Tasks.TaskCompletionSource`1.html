Class TaskCompletionSource<TResult>

Represents the producer side of a System.Threading.Tasks.Task{TResult} unbound to a delegate, providing access to the consumer side through the Task property.

Inheritance
Object

Syntax
public class TaskCompletionSource
Remarks
It is often the case that a System.Threading.Tasks.Task{TResult} is desired to represent another asynchronous operation. TaskCompletionSource is provided for this purpose. It enables the creation of a task that can be handed out to consumers, and those consumers can use the members of the task as they would any other. However, unlike most tasks, the state of a task created by a TaskCompletionSource is controlled explicitly by the methods on TaskCompletionSource. This enables the completion of the external asynchronous operation to be propagated to the underlying Task. The separation also ensures that consumers are not able to transition the state without access to the corresponding TaskCompletionSource. All members of TaskCompletionSource are thread-safe and may be used from multiple threads concurrently.

Task

Gets the System.Threading.Tasks.Task{TResult} created by this TaskCompletionSource.

Remarks

This property enables a consumer access to the System.Threading.Tasks.Task{TResult} that is controlled by this instance. The SetResult(TResult), SetException(Exception), SetException(IEnumerable, and SetCanceled() methods (and their "Try" variants) on this instance all result in the relevant state transitions on this underlying Task.
